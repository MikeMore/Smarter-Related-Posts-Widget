Related Posts Widget for Google Blogger v2.0 - Setup

Demo 1 Demo 2 Demo 3 Demo 4

Customize widget options, and this wizard will demonstrate and generate the code for you.
Click "Add to Blogger" to get redirected to Google Blogger and have code placed in a HTML/JS page element.

Options

Leave inputs empty to use defaults. more info on widget options
  • A Google Blogger Blog URL without the trailing slash. Or leave empty to aggregate yours.
  • Related posts widget or Recent posts widget
  • Fixed list to aggregate like ['jQuery','twitter'] - but note that tags are case-sensitive!
  • Max posts to appear. For recent posts widget this option specify number of posts to return. default is 5.
  • Max tags to look for on the post page. default is 5.
  • Max. posts to be query per tag -this option has no effect on recent posts widget. default is 5.
  • Show posts titles or not.
  • Show posts thumbnails or not.
  • Default image URL to use as thumbnail for posts that don't have one.
  • To retrieve a 72 pixel image that is cropped, you would specify 's72-c' -which is the default, while to retrieve the uncropped image, you would specify 's72'.
    Other accepted cropped and uncropped sizes are:
    32, 48, 64, 72, 104, 144, 150, 160.
    uncropped sizes only:
    94, 110, 128, 200, 220, 288, 320, 400, 512, 576, 640, 720, 800, 912, 1024, 1152, 1280, 1440, 1600.
  • Use one these CSS styles to get started quickly
  • Number of links/images to show per transition. apparently should be less than max_posts. If you set this number to 0 then transitions will be disabled and links are displayed as a simple list.
  • Time in milliseconds for a link to be shown.
  • Entry transition time in milliseconds.
  • Exit transition time in milliseconds.
  • What to animate for entry transition. by default is set to 'opacity' to fade link/thumb in.
  • Show posts thumbnails or not.
  • CSS class will be set on list until widget is loaded. default is 'rpw-loading'
  • h3 Title to be added for related posts widget.
  • h3 Title to be added for recent posts widget.
  • Timeout per request in milliseconds to display widget even if one of the feeds request didn't complete.
    Default is 1500
  • A CSS class prefix to be used to style posts based on relevancy.
    If you set this to 'related-link' and max_tags is 5 then related links can have classes like ('related-link1' to 'related-link5')
  • You can abort widget if current page is not a post page by setting this to 1 which is useful if you don't want widget to appear on stand-alone pages. by default this option is disabled.
  • Any query string in the container page URL will be ignored, which is suitable to Google Blogger platform.
  • Auto-load any number of div elements with "related-posts-widget" class and place options in a HTML comment or HTML5 data attribute. Also, you can use a jQuery method call.

Code

Copy and paste into a HTML/JavaScript Gadget under Page Elements. Or click "Add to blogger" to get code copied to your blog.

Notes

  • Javascript files -jQuery library & Widget- should be included in that order. Each file only needs to exists one time on a page regardless of how many widgets you have on that page.
  • You can include the Javascript files at <head> section or by the end of the page.
  • When setting options in HTML5 attribute, you should escape single-quotes with backslash in options values.
    so, you would write 'John\'s Post' instead of 'John's Post'.
    And escape double-quotes like this 'say "hi"' instead of 'say "hi"'.
  • If you place options in HTML comment, you should escape single-quotes as before. And put a backslash between any 2 consecutive dashes like this 'related -\- Post' instead of 'related -- Post'.
  • Also, If you place options in HTML comment and embed code in Blogger template, you need to escape the HTML comment like this:
    &lt;!-- {
  max_posts:4
  ,max_tags:4
  ,...
} --&gt;
  • Plugin was also tested to work with jQuery 1.3.2, Just replace jQuery Javascript source to "http://ajax.googleapis.com/ajax/libs/jquery/1.3.2/jquery.min.js"
